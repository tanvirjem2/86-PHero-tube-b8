Discuss the scope of var, let, and const

var declarations are globally scoped or function scoped while let and const are block scoped.

var variables can be updated and re-declared within its scope; let variables can be updated but not re-declared; const variables can neither be updated nor re-declared.

They are all hoisted to the top of their scope. But while var variables are initialized with undefined, let and const variables are not initialized.

While var and let can be declared without being initialized, const must be initialized during declaration.

Tell us the use cases of null and undefined

null is a primitive type of a variable which evaluates falsy, has a typeof() of object, and is typically intentionally declared as null

undefined is a primitive type of a variable which evaluates falsy, has a typeof() of undefined, and represents a variable that is declared but missing an initial value.

null == undefined evaluates as true because they are loosely equal.

null === undefined evaluates as false because they are not, in fact, equal.

What do you mean by REST API?

A REST API is like a bridge that allows different computer systems to talk to each other over the internet. It works like a universal language that computers understand.

Imagine it as a digital messenger that helps apps and websites share information. These APIs define how requests and responses should be structured, making it easy for one system to ask for data or perform actions on another system, like fetching weather data from a weather website or posting a photo on social media.

It's a way for computers to collaborate and share data seamlessly, making many of the online services and apps we use every day possible.
